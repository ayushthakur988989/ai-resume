# AI Resume Builder

The project is split into two independent applications:

```text
ai-resume/
|-- frontend/   React + Vite application
|-- backend/    Node.js API
|-- README.md
`-- .gitignore
```

## Run both applications

From the project root:

```bash
npm run dev
```

This starts the frontend and backend together. Press `Ctrl+C` to stop both.

## Frontend

```bash
cd frontend
npm run dev
```

The frontend runs at `http://localhost:5173`.

## Backend

```bash
cd backend
npm run dev
```

The backend runs at `http://localhost:5000`. Its initial health endpoint is `GET /api/health`.

The frontend uses the backend API for authentication and user-scoped resume storage in MongoDB Atlas.

## Production deployment

The repository includes deployment configuration for a Vercel frontend and a
Render backend. Users and resumes are stored in MongoDB Atlas.

### 1. Secure MongoDB Atlas

1. Rotate the current Atlas database-user password before deployment.
2. Give the application user only `readWrite` access to the `ai-resume` database.
3. Restrict Network Access to your host's outbound IP ranges when the selected
   hosting plan provides stable addresses. Do not use an Atlas admin user in the app.

### 2. Deploy the backend on Render

Create a Render Blueprint from this repository. Render reads `render.yaml` and
prompts for these secret values:

- `MONGODB_URI`: the new Atlas connection string.
- `FRONTEND_ORIGIN`: the Vercel URL, without a trailing slash. Multiple allowed
  origins can be separated with commas.

`JWT_SECRET` is generated by Render. Never copy production secrets into `.env`
or commit them to Git.

Confirm the deployed health endpoint returns `database: "connected"`:

```text
https://<render-service>.onrender.com/api/health
```

### 3. Deploy the frontend on Vercel

Import the same repository and configure:

- Root Directory: `frontend`
- Build Command: `npm run build`
- Output Directory: `dist`
- Environment variable `VITE_API_URL`:
  `https://<render-service>.onrender.com/api`

The included `frontend/vercel.json` handles React Router page refreshes.
After Vercel provides the final domain, update `FRONTEND_ORIGIN` on Render and
redeploy the backend.

### Production checklist

- Sign up with a new account and verify the user document in Atlas.
- Create, edit, download, and delete a resume.
- Verify that a second account cannot access the first account's resumes.
- Check `/api/health` and review Render logs.
